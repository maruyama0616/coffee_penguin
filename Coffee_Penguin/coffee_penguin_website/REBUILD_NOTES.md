# Mocha the Coffee Penguin オフィシャルサイト 再構築ノート

## 概要

このドキュメントは、Mocha the Coffee Penguinのオフィシャルサイトを、策定したコンセプト・要件定義・デザイン設計に基づいて再構築した内容をまとめたものです。

## 再構築の方針

### 1. デザインコンセプト: 「静けさのデザイン」

サイト全体を通じて、「デジタル・サードプレイス『喫茶モカ』」というコンセプトを体現しています。訪れるファン一人ひとりにとっての「心の止まり木」となるような、静かで心地よい体験を提供します。

### 2. カラーパレット

以下のカラーパレットを新たに定義し、サイト全体に適用しました：

| カラー名 | HEX | 用途 |
| :--- | :--- | :--- |
| Coffee Dark | `#4A3728` | 見出し、主要テキスト |
| Coffee Medium | `#6B4E3D` | リンク、アクセント |
| Coffee Light | `#8B6F47` | ホバー状態、補助的な要素 |
| Warm Beige | `#E8DCC4` | タグ、カード背景 |
| Cream | `#F5F1E8` | ページ背景 |
| Steel Blue | `#B0C4D8` | アクセント、セクション背景 |
| Steel Blue Light | `#D4E1ED` | 淡いセクション背景 |

これらは `src/app/globals.css` の `:root` セクションでCSS変数として定義されています。

### 3. タイポグラフィ

*   **見出し**: 手書き風フォント（Klee One, Zen Maru Gothic）を使用し、温かみを表現
*   **本文**: システムフォント + Noto Sans JPで、読みやすさを重視
*   **行間**: 1.8に設定し、ゆったりとした読書体験を提供

### 4. スペーシング（余白）

「余白を贅沢に使う」という方針に基づき、以下のスペーシングシステムを導入：

*   `--spacing-xs`: 0.5rem
*   `--spacing-sm`: 1rem
*   `--spacing-md`: 2rem
*   `--spacing-lg`: 4rem
*   `--spacing-xl`: 6rem
*   `--spacing-2xl`: 8rem

## 新規追加ページ

### Web漫画（Stories）ページ

**パス**: `/stories`

Mochaの日常を描いたWeb漫画を一覧表示するページです。「言葉少なめで、感情や仕草で語る」というキャラクターコンセプトを体現したコンテンツの中心となります。

**主な機能**:
*   漫画の一覧表示（グリッドレイアウト）
*   テーマ別のタグ表示
*   公開日の表示
*   サムネイル画像のホバーエフェクト

**ファイル**: `src/app/stories/page.tsx`

### Web漫画詳細ページ

**パス**: `/stories/[slug]`

個別の漫画を閲覧するページです。複数のコマ（パネル）を縦スクロールで読み進める形式です。

**主な機能**:
*   コマ画像の表示
*   ストーリー情報（タイトル、テーマ、公開日、説明文）
*   一覧ページへの戻るリンク
*   他のストーリーへのナビゲーション

**ファイル**: `src/app/stories/[slug]/page.tsx`

## 更新された既存ファイル

### 1. `src/app/globals.css`

デザインシステムのカラーパレット、タイポグラフィ、スペーシングを追加しました。既存のスタイルとの互換性を保つため、レガシーカラー変数も残しています。

**主な追加内容**:
*   Mochaのカラーパレット（CSS変数）
*   スペーシングシステム
*   タイポグラフィ設定
*   アニメーション設定（控えめで優しい）

### 2. `src/components/navigation.tsx`

ナビゲーションメニューに「Stories」リンクを追加しました。

**変更内容**:
*   左側ナビゲーションに `Stories` を追加
*   右側ナビゲーションに `News` を移動してバランスを調整

## 仮画像の配置

以下の仮キャラクター画像を生成し、`public/images/` に配置しました：

| ファイル名 | 用途 |
| :--- | :--- |
| `mocha-hero.png` | ヒーローセクション、メインビジュアル |
| `mocha-profile.png` | プロフィールページ、サイドビュー |

これらの画像ファイルを、実際のキャラクターイラストで上書きするだけで、サイト全体に反映されます。ファイル名を変更する必要はありません。

## 技術スタック

*   **フレームワーク**: Next.js 15 (App Router)
*   **スタイリング**: TailwindCSS 4 + カスタムCSS
*   **データベース**: Prisma + (MySQL/TiDB)
*   **UI コンポーネント**: Radix UI
*   **アニメーション**: Framer Motion
*   **アイコン**: Lucide React

## 開発環境のセットアップ

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev

# ビルド
npm run build

# 本番サーバーの起動
npm start
```

開発サーバーは `http://localhost:3000` で起動します。

## デザインガイドライン

### やるべきこと

*   余白を贅沢に使う
*   アニメーションは控えめで優しく
*   カラーパレットを厳守する
*   手書き風フォントは見出しのみに使用
*   画像は必ず最適化する

### やってはいけないこと

*   派手な色や強いコントラストを使わない
*   急激なアニメーションや点滅を避ける
*   情報を詰め込みすぎない
*   説明しすぎない（余白で語る）
